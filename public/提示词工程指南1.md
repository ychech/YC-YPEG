# <font style="background-color:rgba(255, 255, 255, 0.1);">1、 采样</font>
我们所说的LLM也就是大语言模型 （Large Language Model ）本质就是**<font style="background-color:rgba(255, 255, 255, 0);">下一个词预测器，</font>**它只做一件事：

> 根据前面的文字，**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">预测下一个 token（词 / 字 / 符号）出现的概率</font>**。
>

_如果你想知道具体内容请参考（Attention Is All You Need）_



<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">简单来说你给模型一句话，它每一步都要做一件事：</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">从所有汉字 / 单词里，挑下一个字出来。</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">但它不是瞎挑，而是：</font>

1. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">算出</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">每个词接下来出现的概率</font>**
2. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">按概率</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">随机选一个</font>**

_**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">当然这里不会直接只选概率最大的那个，那样会很呆板、重复，而是根据概率分布随机抽取一个</font>**_

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">这个 “</font>**<font style="color:rgb(0, 0, 0);background-color:#FBDE28;">按概率随机选一个词</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">”的动作，就叫：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">采样（sampling）</font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);"></font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">补充：</font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">只选概率最大的那个，叫 greedy decoding（贪心解码），不算真正意义上的随机采样，只是最最基础的 LLM 生成策略，接下来我们会介绍大模型中几种有约束的随机采样的策略</font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);"></font>**





## 1.1、<font style="background-color:rgba(255, 255, 255, 0.1);">采样阶段的参数</font>
### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.1、 采样参数</font>
<font style="background-color:rgba(255, 255, 255, 0.1);">大模型生成文字的过程，本质就是在 “抽样 / 采样”，所以在这个控制过程中需要的一些参数来调控一些概率,我们叫采样参数。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">采样参数，</font>或者叫<font style="background-color:rgba(255, 255, 255, 0.1);">采样策略，采样算法，我们一般常见常用的有两个Temperature 和Top_p</font>

> <font style="background-color:rgba(255, 255, 255, 0.1);">这个两个采样参数你可以想象成一个人的情绪，当这个人温度比较高很top的时候，你可以类比成比较兴奋的时候，他的动作和语言会多样性，反之一个冷静的人就会比较固定的回复。</font>
>
> <font style="background-color:rgba(255, 255, 255, 0.1);">或者你可以认为是饮酒，当然不同模型差异比较大，比如李白喝酒诗兴大发写出佳作，普通人一般来说可能胡言乱语。</font>

#### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.1.2、</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">Top_p</font>`
**<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">全称</font>**<font style="background-color:rgba(255, 255, 255, 0.1);">：</font>**<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Nucleus Sampling</font>**<font style="background-color:rgba(255, 255, 255, 0.1);">（核采样）</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">准确来说，核采样是一种生成式采样算法，对应的参数是：</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">top_p</font>**

<font style="color:#DF2A3F;background-color:rgba(255, 255, 255, 0.1);">top_p 越高，多样性越高，这个值越小，确定性越强</font>

我们刚刚讨论了 temperature，它也是越高越随机，越低越固定，temperature 是概率来确定的，但是top_p 不一样

<font style="background-color:rgba(255, 255, 255, 0.1);">它是一种基于</font>**<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">概率累积分布</font>**<font style="background-color:rgba(255, 255, 255, 0.1);">的采样策略，核心是设定一个概率阈值 </font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">P</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">，模型会先将所有候选词按概率从高到低排序，然后累加其概率值，直到累加和≥</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">P</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">，最后只在这个 “核心词集合” 中随机选择下一个词。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">用Top P意味着只有词元集合（tokens）中包含top_p概率质量的才会被考虑用于响应，因此较低的top_p值会选择最有信心的响应。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">这意味着较高的top_p值将使模型考虑更多可能的词语，包括不太可能的词语，从而导致更多样化的输出。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);"></font>

核采样，顾名思义，只采取核心，意思就是只取概率分布里最核心、最靠谱、概率最高的那一小堆词

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">只保留 “概率加起来 ≥ p” 的最小词集合，其他全部扔掉。</font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);"></font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">如果说到这里你还有点不清楚，我们可以拿刚刚的示例举例子</font>

```python
今天天气很____
```

<font style="background-color:rgba(255, 255, 255, 0.1);">还是刚刚的概率</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">好 (80%)、棒 (10%)、热 (5%)、冷 (3%)、糟 (2%)</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">如果 top_p = 0.9</font>

+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">好：0.8   此时</font><font style="background-color:rgba(255, 255, 255, 0.1);">top_p = 0.9，</font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">0.8 < 0.9 继续</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">棒：0.1   此时好的概率+棒的概率  累计 0.9 ≥ 0.9</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">所有确定</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">核 = {好，棒}</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">外面的词：热</font><font style="background-color:rgba(255, 255, 255, 0.1);">(5%)</font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">、冷</font><font style="background-color:rgba(255, 255, 255, 0.1);">(3%)</font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">、糟 </font><font style="background-color:rgba(255, 255, 255, 0.1);">(2%)</font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">→ 直接</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">扔掉</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">。</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);"></font>

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">再把核里的概率</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">重新  归一化</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">（概率总和变 100%，就是 1）：</font>

+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">好：0.8 / 0.9 ≈ </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">88.9%</font>**
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">棒：0.1 / 0.9 ≈ </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">11.1%</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">只在这两个里选</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">所以本质上</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">top_p 数值越高，候选词越多，输出越随机，候选词，越大，选的词就越多,反之就是越固定</font>



+ `<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Top-P = 1</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">：等同于不限制，包含所有候选词，随机性最大。</font>
+ `<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Top-P = 0.7</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">：只保留概率累加和≥70% 的核心词，过滤低概率生僻词，保证输出流畅性，同时保留一定多样性。</font>
+ `<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Top-P 趋近于 0</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">：只保留概率最高的极少数词，输出极度确定，类似 </font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Temperature=0</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">。</font>

#### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.1.3、一些注意点</font>
<font style="background-color:rgba(255, 255, 255, 0.1);">注意1：一般建议是改变 Temperature 和 Top P 其中一个参数就行，不用两个都调整，另一个调整方式：改回默认值</font>`<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">1.0</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">（即不限制候选词范围）。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">注意2:</font>**<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">与 Temperature 的区别</font>**

+ `<font style="color:rgb(31, 35, 41);background-color:rgba(0, 0, 0, 0);">Top-P</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">：通过截断候选词集合控制随机性，不改变词的原始概率占比，更聚焦高概率词。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">注意3：如果在开发中，实际业务没有问题，实际效果没问题的话，不用纠结理论，可以不修改。</font>

### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.3、惩罚参数</font>
#### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.3.1、</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">Frequency_Penalty</font>`
<font style="background-color:rgba(255, 255, 255, 0.1);">中文名是叫：频率惩罚</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">对于惩罚参数，它同样是在采样之前进行处理，</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">之前我们已经说明了大模型本质是预测生成下一个单词/内容</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">大语言模型生成文本是</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">逐 token 选最优解</font>**<font style="background-color:rgba(255, 255, 255, 0.1);">的过程：</font>

+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">每生成一个词（token），模型会先计算</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">词汇表中所有 token 的 “得分”（概率）</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">；这个</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">得分</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">。（模型输出的 </font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">logits（对数几率）</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">，它</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">不是概率</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">，但可以变成概率）</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">得分最高的 token 会被选为下一个输出；</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">频率惩罚的核心，就是</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">修改这些 token 的得分</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">，让 “已经出现过的 token” 得分变低，从而降低它被选中的概率。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">线性累进惩罚</font>**（出现 1 次罚 1 份，出现 3 次罚 3 份）adjusted_logit=original_logit−frequency_penalty×count

最后得分=模型原本给这个 token 的原始分数-你设置的惩罚系数（如 0.5、1.0）*这个 token **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">已经出现的次数</font>**

<font style="background-color:rgba(255, 255, 255, 0.1);">这个惩罚和 token 在响应和提示中已出现的次数成比例， </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">frequency_penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 越高，某个词再次出现的可能性就越小，这个设置通过给 重复数量多的 Token 设置更高的惩罚来减少响应中单词的重复。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);"></font>

#### <font style="background-color:rgba(255, 255, 255, 0.1);">1.1.3.2、</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">Presence_Penalty</font>`
<font style="background-color:rgba(255, 255, 255, 0.1);">存在惩罚</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">只看有没有重复</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">出现两次的 token 和出现 10 次的 token 会受到相同的惩罚。</font>

`<font style="background-color:rgba(255, 255, 255, 0.1);">presence_penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 也是对重复的 token 施加惩罚，但与 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">frequency penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 不同的是，惩罚对于所有重复 token 都是相同的。</font>

+ **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">Frequency Penalty</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">：按</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">次数</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">罚，次数越多罚越重   </font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">−α×count</font>
+ **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">Presence Penalty</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">：只要</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">出现过就罚一次</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">，和次数无关 </font><font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">−β×1</font>

<font style="background-color:rgba(255, 255, 255, 0.1);"> 此设置可防止模型在响应中过于频繁地生成重复的词。 如果您希望模型生成多样化或创造性的文本，可以设置更高的 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">presence penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">，如果希望模型生成更专注的内容，您可以设置更低的 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">presence penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);">。</font>

<font style="background-color:rgba(255, 255, 255, 0.1);">与 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">temperature</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 和 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">top_p</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 一样，一般建议是改变 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">frequency penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 和 </font>`<font style="background-color:rgba(255, 255, 255, 0.1);">presence penalty</font>`<font style="background-color:rgba(255, 255, 255, 0.1);"> 其中一个参数就行，不要同时调整两个。</font>

                ↓

### <font style="color:rgb(0, 0, 0);">2.1.1、RepetitionPenalty</font>
<font style="color:rgb(0, 0, 0);">解决刷屏式重复，强力压制高频重复</font>

<font style="color:rgb(0, 0, 0);">日常用 1.1~1.3</font>

但是出现 1 次和出现 10 次，惩罚**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一样大</font>**

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">核心逻辑：用除法降低重复 Token 的分数，分数越低 → 被选中的概率越低。</font>**

+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">取值 = 0.0：该惩罚不生效；</font>
+ <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">取值越高：惩罚越重（但超过 2.0 容易导致生成内容逻辑混乱）。</font>

<font style="color:rgb(0, 0, 0);">和 Presence/FrequencyPenalty 不同，它用除法而非减法，惩罚更温和、更可控，是解决重复问题的首选参数，也比较常用，简单粗暴防重复。如果需要精确的处理，PresencePenalty + FrequencyPenalty 我们上面已经提过这里不再重复。</font>

## <font style="color:#000000;background-color:rgba(255, 255, 255, 0);">3.1、GPT</font>
<font style="color:#000000;background-color:rgba(255, 255, 255, 0);"> 在 Responses API 中 OpenAI 推荐的新一代文本生成 API，主要面向 GPT-5、o3 等</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">推理模型（Reasoning models）</font>**<font style="color:#000000;background-color:rgba(255, 255, 255, 0);"> 设计，其核心变化是</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">弱化了手动配置采样参数</font>**<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">：</font>

<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">对于 GPT-5.4、GPT-5 这类新模型，OpenAI 将</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">temperature</font>`<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">、</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">top_p</font>`<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">等采样逻辑</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">内置优化</font>**<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">，默认根据任务类型（如创意生成、事实问答）自动适配</font>

<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">一般像推理等固定化场景的模型，都没有办法调整不支持，chat 一般支持</font>

<font style="color:#000000;background-color:rgba(255, 255, 255, 0);">但是如果想精细化调优请参考</font>

[https://developers.openai.com/api/reference/resources/responses/methods/create](https://developers.openai.com/api/reference/resources/responses/methods/create)
