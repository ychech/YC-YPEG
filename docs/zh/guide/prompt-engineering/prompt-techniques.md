# 常用提示技术（零样本 / 少样本 / CoT 等）

提示技术的本质是：通过更好的“问题组织方式”，让模型在相同能力下输出更靠谱的结果。

## 1、零样本提示（Zero-shot）

不给示例，直接下指令。适合：

- 规则明确
- 输出格式明确
- 任务复杂度中等

建议把“输出格式”和“禁止项”写清楚，避免模型发挥。

## 2、少样本提示（Few-shot）

给 1～5 组示例，让模型学会你想要的风格与格式。适合：

- 风格很重要（文案/客服话术/摘要）
- 你要模型模仿结构

注意示例要覆盖边界情况，否则模型会只学到“顺风局”。

## 3、链式思考（CoT）

CoT 的价值是让模型把推理过程显式化，更利于发现错误、提高复杂问题成功率。

在工程里，建议用“两段式”：

1. 先让模型输出推理草稿（仅用于内部）
2. 再让模型输出最终答案（面向用户）

如果你担心暴露推理过程，可以要求模型只输出结论与关键依据。

## 4、自洽性（Self-consistency）

让模型在同一提示下多次生成，然后投票/聚合。适合：

- 推理题
- 多步计算
- 需要降低偶发错误

成本会上升，但稳定性也会上升。

## 5、分解与规划（Decompose / Plan）

当任务复杂时，先拆问题，再逐步完成：

- 先列出子任务清单
- 再逐项完成
- 最后汇总输出

这类方式对“长文本改写/信息抽取/多约束生成”很有效。
