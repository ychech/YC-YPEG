# 1、 采样

我们所说的LLM也就是大语言模型 （Large Language Model ）本质就是**下一个词预测器，**它只做一件事：

> 根据前面的文字，**预测下一个 token（词 / 字 / 符号）出现的概率**。

_如果你想知道具体内容请参考（Attention Is All You Need）_

简单来说你给模型一句话，它每一步都要做一件事：

**从所有汉字 / 单词里，挑下一个字出来。**但它不是瞎挑，而是：

1. 算出**每个词接下来出现的概率**
2. 按概率**随机选一个**

_**当然这里不会直接只选概率最大的那个，那样会很呆板、重复，而是根据概率分布随机抽取一个**_

这个 “**按概率随机选一个词**”的动作，就叫：**采样（sampling）**

**补充：**

**只选概率最大的那个，叫 greedy decoding（贪心解码），不算真正意义上的随机采样，只是最最基础的 LLM 生成策略，接下来我们会介绍大模型中几种有约束的随机采样的策略**

## 1.1、采样阶段的参数
### 1.1.1、 采样参数

大模型生成文字的过程，本质就是在 “抽样 / 采样”，所以在这个控制过程中需要的一些参数来调控一些概率,我们叫采样参数。

采样参数，或者叫采样策略，采样算法，我们一般常见常用的有两个:Temperature 和Top_p

> 这个两个采样参数你可以想象成一个人的情绪，当这个人温度比较高很top的时候，你可以类比成比较兴奋的时候，他的动作和语言会多样性，反之一个冷静的人就会比较固定的回复。
>
> 或者你可以认为是饮酒，当然不同模型差异比较大，比如李白喝酒诗兴大发写出佳作，普通人一般来说可能胡言乱语。

#### 1.1.1.1、`Temperature`
中文叫温度

温度越高，多样性越高，这个值越小，确定性越强

详细的来说 temperature

temperature 作用是 “软化概率分布”，是一种概率控制

——让数值越高，输出越随机；数值越低，输出越确定。

如下

| 高↑  | 随机 |
| --- | --- |
| 低 ↓ | 确定 |

软化是相对于“硬” 的概率分布的，

假如对于

```python
今天天气很____
“硬” 的概率分布是（Temperature = 0）

| 词语 | 概率 | 选中可能性 |
| --- | --- | --- |
| 好 | 80% | 几乎必选 |
| 棒 | 10% | 极少出现 |
| 热 | 5% | 几乎不出现 |
| 冷 | 3% | 几乎不出现 |
| 糟 | 2% | 几乎不出现 |

好  : ████████████████████ 80%
棒  : ██                  10%
热  : █                   5%
冷  : ▌                   3%
糟  : ▌                   2%

那么（Temperature = 2.0（极软，完全被拉平））

| 词 | 概率 | 选中可能性 |
| --- | --- | --- |
| 好 | 30% | 可能出现 |
| 棒 | 25% | 可能出现 |
| 热 | 20% | 可能出现 |
| 冷 | 15% | 可能出现 |
| 糟 | 10% | 可能出现 |

好  : ██████              30%
棒  : █████               25%
热  : ████                20%
冷  : ███                 15%
糟  : ██                  10%
可以看到 Temperature ↑：不断软化小概率词被抬升，分布越来越平，输出随机、多样、 creative。比如诗歌等创造性的，可以适当调大。

常用的

- Temperature = 0：生成结果最确定，模型始终选择概率最高的词，输出严谨但可能重复。
- Temperature > 1：随机性增强，生成内容更多样、有创意，但可能出现逻辑混乱。
- Temperature < 1：介于两者之间，平衡确定性与多样性。

#### 1.1.1.2、Top_p
全称：Nucleus Sampling（核采样）

准确来说，核采样是一种生成式采样算法，对应的参数是：top_p

top_p 越高，多样性越高，这个值越小，确定性越强

我们刚刚讨论了 temperature，它也是越高越随机，越低越固定，temperature 是概率来确定的，但是top_p 不一样

它是一种基于概率累积分布的采样策略，核心是设定一个概率阈值 P，模型会先将所有候选词按概率从高到低排序，然后累加其概率值，直到累加和≥P，最后只在这个 “核心词集合” 中随机选择下一个词。

用Top P意味着只有词元集合（tokens）中包含top_p概率质量的才会被考虑用于响应，因此较低的top_p值会选择最有信心的响应。

这意味着较高的top_p值将使模型考虑更多可能的词语，包括不太可能的词语，从而导致更多样化的输出。

核采样，顾名思义，只采取核心，意思就是只取概率分布里最核心、最靠谱、概率最高的那一小堆词

只保留 “概率加起来 ≥ p” 的最小词集合，其他全部扔掉。

如果说到这里你还有点不清楚，我们可以拿刚刚的示例举例子

今天天气很____

还是刚刚的概率

好 (80%)、棒 (10%)、热 (5%)、冷 (3%)、糟 (2%)

如果 top_p = 0.9

+ 好：0.8   此时top_p = 0.9，0.8 < 0.9 继续
+ 棒：0.1   此时好的概率+棒的概率  累计 0.9 ≥ 0.9
+ 所有确定核 = {好，棒}
+ 外面的词：热(5%)、冷(3%)、糟 (2%)→ 直接扔掉。

再把核里的概率重新  归一化（概率总和变 100%，就是 1）：

+ 好：0.8 / 0.9 ≈ 88.9%
+ 棒：0.1 / 0.9 ≈ 11.1%

只在这两个里选

所以本质上

top_p 数值越高，候选词越多，输出越随机，候选词，越大，选的词就越多,反之就是越固定

+ Top-P = 1：等同于不限制，包含所有候选词，随机性最大。
+ Top-P = 0.7：只保留概率累加和≥70% 的核心词，过滤低概率生僻词，保证输出流畅性，同时保留一定多样性。
+ Top-P 趋近于 0：只保留概率最高的极少数词，输出极度确定，类似 Temperature=0。

#### 1.1.1.3、一些注意点
注意1：一般建议是改变 Temperature 和 Top P 其中一个参数就行，不用两个都调整，另一个调整方式：改回默认值1.0（即不限制候选词范围）。

注意2:与 Temperature 的区别

+ Temperature：通过缩放概率分布调整随机性，全局改变所有词的概率权重。
+ Top-P：通过截断候选词集合控制随机性，不改变词的原始概率占比，更聚焦高概率词。

注意3：如果在开发中，实际业务没有问题，实际效果没问题的话，不用纠结理论，可以不修改。

### 1.1.2、 终止参数
#### 1.1.2.1、Max_Length
指定 Max Length 有助于防止大模型生成冗长或不相关的响应并控制成本。

单位：token

这里需要注意的是，Max Length 是按 token 算，不是按字算；

Token = 模型读写文字的「最小单位」

今天天气很好

在大部分模型来看

+ 今 → 1 个 token
+ 天 → 1 个 token
+ 天 → 1 个 token
+ 气 → 1 个 token
+ 很 → 1 个 token
+ 好 → 1 个 token

但是也可能

今天→ 1 个 token
天气→ 1 个 token

只不过大部分情况情况下中文里 1 字 ≈ 1 token

英文同样如此
beautiful→ 常见 → 整块 1 token

这里Token 是由模型自带的一个工具 Tokenizer（分词器） 切出来的。

### 1.1.3、惩罚参数
#### 1.1.3.1、Frequency_Penalty
中文名是叫：频率惩罚

对于惩罚参数，它同样是在采样之前进行处理，

之前我们已经说明了大语言模型本质是预测生成下一个单词/内容

大语言模型生成文本是逐 token 选最优解的过程：

+ 每生成一个词（token），模型会先计算词汇表中所有 token 的 “得分”（概率）；这个得分。（模型输出的 logits（对数几率），它不是概率，但可以变成概率）
+ 得分最高的 token 会被选为下一个输出；
+ 频率惩罚的核心，就是修改这些 token 的得分，让 “已经出现过的 token” 得分变低，从而降低它被选中的概率。

线性累进惩罚（出现 1 次罚 1 份，出现 3 次罚 3 份）adjusted_logit=original_logit−frequency_penalty×count

最后得分=模型原本给这个 token 的原始分数-你设置的惩罚系数（如 0.5、1.0）*这个 token 已经出现的次数

这个惩罚和 token 在响应中已出现的次数成比例， frequency_penalty 越高，某个词再次出现的可能性就越小，这个设置通过给 重复数量多的 Token 设置更高的惩罚来减少响应中单词的重复。

#### 1.1.3.2、Presence_Penalty
存在惩罚

只看有没有重复

出现两次的 token 和出现 10 次的 token 会受到相同的惩罚。

presence_penalty 也是对重复的 token 施加惩罚，但与 frequency penalty 不同的是，惩罚对于所有重复 token 都是相同的。

+ Frequency Penalty：按次数罚，次数越多罚越重   −α×count
+ Presence Penalty：只要出现过就罚一次，和次数无关 −β×1

此设置可防止模型在响应中过于频繁地生成重复的词。 如果您希望模型生成多样化或创造性的文本，可以设置更高的 presence penalty，如果希望模型生成更专注的内容，您可以设置更低的 presence penalty。

与 temperature 和 top_p 一样，一般建议是改变 frequency penalty 和 presence penalty 其中一个参数就行，不要同时调整两个。

⚠️注意：请记住最终生成的结果可能会和使用的大语言模型的版本而异，在实践过程中需要不断调试并且优化

[105, 349, 458]

模型（Transformer）算一遍

原始 Logits（50257 个分数）

[ 分数0, 分数1, 分数2, ...分数33...分数5677.., 分数50256 ]

[0.83,   8.36, .. 8.71, ..., 9.64, .....27.64 .. ........0.11.]

对原始 Logits 可以进行如下流程操作

模型输出 → 原始 Logits
                ↓
[LogitBias] → 加减 logits（如：+10 强制选择某个 token）
                ↓
[RepetitionPenalty] → 乘除 logits（惩罚已出现过的 token）
                ↓
[PresencePenalty] → 减 logits（只要出现过，就减去固定值）
                ↓
[FrequencyPenalty] → 减 logits × 出现次数（出现越多，减越多）
                ↓
[Temperature] → logits / T
                ↓
Softmax → 概率分布
                ↓
[Top-K] → 截断
    ↓
[Top-P] → 截断
    ↓
采样

## 2.1、 补充
原始 Logits 中，“饭”（ID567）的分数是 27.64，“肉”（ID33）的分数是 9.64,；

如果你想让模型 “优先选肉，不选饭”，就可以设置：

LogitBias 给 “肉”（ID33）加 10 → 9.64, + 10 = 19.64,；

LogitBias 给 “饭”（ID567）减 30 → 27.64 - 30 = -2.36；

模型几乎一定会选 “肉”，而非 “饭”。

一般使用 LogitBias场景

 1、强制生成指定字 2、引导生成风格 3、屏蔽敏感词 / 无用词 

### 2.1.1、RepetitionPenalty
解决刷屏式重复，强力压制高频重复

日常用 1.1~1.3

但是出现 1 次和出现 10 次，惩罚一样大

核心逻辑：用除法降低重复 Token 的分数，分数越低 → 被选中的概率越低。

+ 取值 = 0.0：该惩罚不生效；
+ 取值越高：惩罚越重（但超过 2.0 容易导致生成内容逻辑混乱）。

和 Presence/FrequencyPenalty 不同，它用除法而非减法，惩罚更温和、更可控，是解决重复问题的首选参数，也比较常用，简单粗暴防重复。如果需要精确的处理，PresencePenalty + FrequencyPenalty 我们上面已经提过这里不再重复。

### 2.1.2、Softmax
Softmax 做的事：全部转成 0~1 的概率，加起来 = 1

也就是 原始 logits 向量：z=[z1, z2, …, zn]

最终z1+z2, …, +zn=1

e：自然常数（≈2.71828）

y=[0.0016, 0.0002, 0.0002, 0.9237, 0.0743]

### 2.1.3、Top-K
Top-K：只留最高分 K 个，其余拉黑。

Top-P：只留概率累计到 P 的最小子集，其余拉黑。

简单的说一个是个数，应该是累加到的概率
