# 提示词工程指南 1：文本生成控制参数

> 提示工程（Prompt Engineering）是一门较新的学科，关注提示词开发和优化，有助于用户将大语言模型（Large Language Model, LLM）用于各场景和研究领域。

---

## 一、文本生成控制参数

我们在写提示词过程中需要调整一些参数去配合我们的测试，这些基础的参数在业内常用的叫法或者说更标准的一些叫法是：

- **生成超参数**（Generation Hyperparameters）
- **解码参数**（Decoding Parameters）

当然怎么叫并不重要，你可以就叫它是控制大模型生成内容的参数，简单说是 **控制参数** 也没问题。

现在，让我们正式进入正题。首先我们需要认识几个名词。

### 1、 采样

我们所说的 LLM 也就是大语言模型（Large Language Model）本质就是**下一个词预测器**，它只做一件事：

> 根据前面的文字，**预测下一个 token（词 / 字 / 符号）出现的概率**。
> *(如果你想知道具体内容请参考 Attention Is All You Need)*

简单来说你给模型一句话，它每一步都要做一件事：**从所有汉字/单词里，挑下一个字出来。**
但它不是瞎挑，而是：
1. 算出**每个词接下来出现的概率**
2. 按概率**随机选一个**

*(当然这里不会直接只选概率最大的那个，那样会很呆板、重复，而是根据概率分布随机抽取一个)*

这个"**按概率随机选一个词**"的动作，就叫：**采样（sampling）**。

**补充：**
只选概率最大的那个，叫 greedy decoding（贪心解码），不算真正意义上的随机采样，只是最基础的 LLM 生成策略。接下来我们会介绍大模型中几种有约束的随机采样的策略。

---

## 1.1、采样阶段的参数

### 1.1.1、 采样参数

大模型生成文字的过程，本质就是在"抽样/采样"，所以在这个控制过程中需要一些参数来调控概率，我们叫采样参数。

采样参数，或者叫采样策略、采样算法，我们一般常见常用的有两个：**Temperature** 和 **Top_p**。

> 这两个采样参数你可以想象成一个人的情绪，当这个人温度比较高很 top 的时候，你可以类比成比较兴奋的时候，他的动作和语言会多样性；反之一个冷静的人就会有比较固定的回复。
> 或者你可以认为是饮酒，当然不同模型差异比较大，比如李白喝酒诗兴大发写出佳作，普通人一般来说可能胡言乱语。

#### 1.1.1.1、 `Temperature`

中文叫**温度**。

**温度越高，多样性越高；这个值越小，确定性越强。**

详细来说：
Temperature 的作用是"软化概率分布"，是一种概率控制——让数值越高，输出越随机；数值越低，输出越确定。

| Temperature | 效果 |
| --- | --- |
| **高 ↑** | 随机、多样 |
| **低 ↓** | 确定、严谨 |

**举个例子：**
假设对于句子：`今天天气很____`

**"硬"的概率分布（Temperature = 0）：**
- 好 (80%) —— 几乎必选
- 棒 (10%) —— 极少出现
- 热 (5%)  —— 几乎不出现
- 冷 (3%)  —— 几乎不出现
- 糟 (2%)  —— 几乎不出现

**"软化"后的概率分布（Temperature = 2.0，极软，被拉平）：**
- 好 (30%) —— 可能出现
- 棒 (25%) —— 可能出现
- 热 (20%) —— 可能出现
- 冷 (15%) —— 可能出现
- 糟 (10%) —— 可能出现

可以看到，随着 Temperature 升高，小概率词被抬升，分布越来越平，输出变得随机、多样、creative。比如写诗歌等创造性任务时，可以适当调大。

**常用设置：**
- `Temperature = 0`：生成结果最确定，模型始终选择概率最高的词，输出严谨但可能重复。
- `Temperature > 1`：随机性增强，生成内容更多样、有创意，但可能出现逻辑混乱。
- `Temperature < 1`：介于两者之间，平衡确定性与多样性。

---

#### 1.1.1.2、 `Top_p`

**全称**：**Nucleus Sampling**（核采样）

准确来说，核采样是一种生成式采样算法，对应的参数是 `top_p`。

**top_p 越高，多样性越高；这个值越小，确定性越强。**

我们刚刚讨论了 temperature，它是靠概率分布的软硬来确定的。但是 top_p 不一样，它是一种基于**概率累积分布**的采样策略。

核心是设定一个概率阈值 `P`，模型会先将所有候选词按概率从高到低排序，然后累加其概率值，直到累加和 `≥ P`，最后只在这个"核心词集合"中随机选择下一个词。
这意味着较高的 top_p 值将使模型考虑更多可能的词语，包括不太可能的词语，从而导致更多样化的输出。较低的 top_p 值会选择最有信心的响应。

核采样，顾名思义，只采取核心，意思就是只取概率分布里最核心、最靠谱、概率最高的那一小堆词。
**只保留"概率加起来 ≥ p"的最小词集合，其他全部扔掉。**

**举个例子：**
还是刚刚的概率：好 (80%)、棒 (10%)、热 (5%)、冷 (3%)、糟 (2%)。

如果设置 `top_p = 0.9`：
1. **好：0.8** (0.8 < 0.9，继续)
2. **棒：0.1** (好的概率 0.8 + 棒的概率 0.1 = 0.9，满足 ≥ 0.9)
3. 确定**核心集合 = {好，棒}**
4. 外面的词（热、冷、糟）直接**扔掉**。

再把核心集合里的概率重新**归一化**（概率总和变 100%）：
- 好：0.8 / 0.9 ≈ **88.9%**
- 棒：0.1 / 0.9 ≈ **11.1%**
模型只会在这两个词里随机选。

**常用设置：**
- `Top-P = 1`：等同于不限制，包含所有候选词，随机性最大。
- `Top-P = 0.7`：只保留概率累加和 ≥ 70% 的核心词，过滤低概率生僻词，保证输出流畅性，同时保留一定多样性。
- `Top-P 趋近于 0`：只保留概率最高的极少数词，输出极度确定，类似 `Temperature=0`。

#### 1.1.1.3、 一些注意点
**注意 1**：一般建议是改变 Temperature 和 Top P 其中一个参数就行，不用两个都调整。另一个参数保持默认值 `1.0`（即不限制候选词范围）。
