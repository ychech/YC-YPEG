# 提示词结构（角色 / 指令 / 约束 / 输出）

提示词的目标不是“把模型说服”，而是把任务的输入、边界、输出格式说清楚，让模型在可控范围内稳定地产出。

## 1、最小可用结构（四要素）

1. 角色（Role）：你希望模型以什么身份解决问题
2. 指令（Task）：要完成的任务是什么
3. 约束（Constraints）：必须遵守的规则、范围、禁止项
4. 输出（Output）：输出格式、字段、语言、长度、风格

你可以把它理解为：让模型知道“你是谁、要做什么、不能做什么、怎么交付”。

## 2、一个可复用的模板

```text
角色：
你是……

任务：
请你……

上下文：
这里是必要背景/资料……

约束：
1) …
2) …

输出格式：
请输出为……

验收标准：
满足……
```

## 3、常见写法与坑

### 3.1 角色不是越多越好

角色用于设定能力边界与表达风格，不要堆叠冲突角色（例如“你既是严格法官又是随意吐槽博主”）。

### 3.2 约束要可执行

“请更专业一点”太抽象；更可执行的写法是：

- 引用来源/证据（如有）
- 禁止编造（不知道就说不知道）
- 结构化输出（标题/列表/JSON）
- 长度限制（例如 200～300 字）

### 3.3 输出格式要写到字段层面

如果你需要结构化结果，不要只说“给我表格/JSON”，而是明确字段：

```json
{
  "title": "",
  "summary": "",
  "key_points": [],
  "risks": []
}
```

## 4、把“稳定”放在第一位

提示词写得再漂亮，如果没有稳定输出，工程上就是不可用。建议你每个提示词都配一份小型评估集：

- 10～30 条典型输入
- 3～5 条刁钻边界输入
- 3～5 条噪声输入（缺字段、错字段、无关内容）

然后用同一模型、同一参数反复回归，确保改动不会把质量搞崩。
