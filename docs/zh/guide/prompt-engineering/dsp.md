# DSP：方向性刺激提示（Directional Stimulus Prompting）

DSP 的核心思想是：给模型一些“方向性线索”（关键词、要点、结构提示），显著提升摘要、改写、信息提取等任务的贴合度与稳定性。

你可以把 DSP 理解为：先把“该往哪写”这件事说清楚，再让模型去发挥。

## 1、适用场景

- 摘要：要求覆盖特定要点
- 改写：要求保留某些信息点，避免跑题
- 抽取：要求按字段输出、避免漏项

## 2、一个实用模板

```text
任务：请根据输入文本生成摘要

方向性刺激（必须覆盖的要点）：
- 要点1：……
- 要点2：……
- 要点3：……

约束：
- 不添加输入中不存在的事实
- 不要展开过度解释

输出：
- 先输出 1 段摘要（100～150 字）
- 再输出要点列表（3～5 条）
```

## 3、为什么有效

当任务很开放时，模型会优先选择“最可能的泛化写法”，导致：

- 漏掉你真正关心的点
- 输出看起来合理但没用

方向性刺激把注意力拉回到你要的要点上，等于在采样空间里做“软约束”。

## 4、工程建议

- 刺激要点不要太多（3～7 条最常用）
- 刺激要点要可验证（能在原文找到证据）
- 与评估集配合使用，否则很难证明有效
