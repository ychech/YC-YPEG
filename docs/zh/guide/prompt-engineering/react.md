# ReAct：思考 + 行动（工具调用）

链式思考（CoT）让模型会“想”，但它仍然无法可靠获取外部事实。ReAct（Reason + Act）解决的是：让模型在推理过程中，按需调用工具获取信息，再继续推理。

## 1、什么时候用 ReAct

- 需要联网/查库/查业务系统
- 需要执行函数（计算、下单、写入系统）
- 需要把复杂流程拆成可执行步骤

## 2、核心结构

一个常见的 ReAct 回合包含：

- Thought：当前怎么想（可内部）
- Action：要调用哪个工具
- Action Input：工具参数
- Observation：工具返回结果

然后进入下一轮，直到输出 Final Answer。

## 3、工程落地要点

### 3.1 工具说明要“像接口文档”

对于每个工具，必须写清楚：

- 工具名称（固定）
- 入参字段（类型/示例）
- 出参字段（类型/示例）
- 错误场景（缺参/超时/权限）

### 3.2 限制动作空间

工具越多、权限越大，越需要约束：

- 白名单工具
- 最大调用次数
- 最大 token / 最大输出长度
- 失败重试策略

### 3.3 输出与执行解耦

建议把“模型输出”和“系统执行”分离：

1. 模型只产生结构化动作（JSON）
2. 系统侧校验动作合法性
3. 系统执行并返回 Observation

这样可控性最好。
