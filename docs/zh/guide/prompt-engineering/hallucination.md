# 模型幻觉与校验

模型幻觉指模型生成了与事实不符、逻辑断裂、脱离上下文的内容，但它往往“非常像真的”。工程上要做的是：降低幻觉发生概率 + 让幻觉可被发现/可被兜底。

## 1、为什么会幻觉

- 模型是概率生成：它在“最可能”而不是“最真实”之间做选择
- 输入信息不足：模型倾向于补全空白
- 任务过于开放：没有边界就会自由发挥

## 2、幻觉的典型形态

- 编造具体细节：人名、论文、版本号、机构
- 引用不存在的链接/来源
- 逻辑链条断裂但语言很流畅

## 3、减少幻觉的提示策略

- 明确“不可编造”：不知道就说不知道
- 要求给出依据：引用原文片段/字段来源
- 强制结构化：把事实与推断分开输出

示例：

```text
请输出两部分：
1) 可证事实：必须能从输入文本中找到对应句子（请附引用）
2) 推断与建议：必须标注为推断，不得伪装成事实
```

## 4、三角验证（强烈推荐）

同一问题用三种方式交叉校验：

- 多模型交叉：不同模型回答对比
- 工具/权威来源：用搜索/数据库核对关键事实
- 人工抽检：抽查高风险问题（法律、医疗、财务）

## 5、技术兜底

- RAG：用检索证据约束回答范围
- 输出校验：正则/JSON schema/字段校验
- 置信度阈值：低置信度转人工/给出澄清问题
